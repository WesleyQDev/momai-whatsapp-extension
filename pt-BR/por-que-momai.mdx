## Propósito

MomAI é opensource e possui suporte para Windows, Linux e Mac ela consegue:

- Captar informações e armazenar em um banco de dados local;
- Executar tarefas quando um evento acontecer;
- Agendar e criar lembretes;
  - Quando chegar a hora marcada do lembrete realizar um aviso por notificação e voz;
- Atender a comandos de voz;

- Interagir com as extensões podendo realizar tarefas em outros sistemas;

<Warning>
  Por padrão, os dados são armazenados localmente e processados pelos provedores
  de LLM. Cabe ao usuário escolher quais provedores utilizar. <br />A MomAI
  possui uma extensão própria que adiciona o `llama.cpp`, permitindo baixar
  modelos locais e executá-los diretamente na assistente. <br />
  No entanto, você também pode optar por usar chaves de API de terceiros.

Extensões podem enviar ou receber informações de serviços externos. <br />{" "}

acesso e certifique-se de que você concorda com os riscos envolvidos.

</Warning>
### Arquitetura

MomAI usa uma arquitetura agêntica, MomAgent é o agente principal que verifica a solicitação do usuário, enquanto os demais agentes são especializados em tarefas específicas, como busca na internet, criação de agendamentos ou criação de interfaces.

Quando o usuário fala, primeiro é analisado por MomAgent, ele atua como um controlador e supervisor dos demais agentes, você pode pensar como um gerente que delega tarefas aos funcionários. Lembre-se disso, cada agente possui um ou mais LLM Providers, prompts, ferramentas e MCPs. Com essa arquitetura é possível desenvolver agentes que possuam ferramentas que se conectam praticamente a qualquer serviço MCP ou API, esses agentes adicionais são disponibilizados por meio de extensões.

<AccordionGroup>
  <Accordion title="Agents">
    Entidades de software autônomas que utilizam modelos de linguagem para
    raciocinar, planejar e executar tarefas ou coordenar outros sistemas,
    atuando como "funcionários" especializados ou "gerentes" supervisores.
  </Accordion>
  <Accordion title="Tools ou ferramentas">
    Funções executáveis ou capacidades específicas que permitem ao agente ir
    além da geração de texto e interagir ativamente com o ambiente, como navegar
    na web, enviar e-mails ou calcular dados.
  </Accordion>
  <Accordion title="Model Context Protocol (MCP)">
    Um padrão aberto que funciona como um "plugue universal", permitindo que a
    IA se conecte facilmente aos seus dados e ferramentas sem a necessidade de
    criar integrações complexas para cada sistema.
  </Accordion>
  <Accordion title="LM Provider ou fornecedor de LLM">
    LM Provider (Fornecedor de LLM): É a infraestrutura que hospeda a
    inteligência do agente, permitindo optar pela execução local para manter a
    privacidade dos dados ou conectar-se a provedores que oferecem APIs
    gratuitas, como Google AI Studio, Groq e OpenRouter.
  </Accordion>
  <Accordion title="API">
    Um conjunto de protocolos e regras de código que permite que softwares
    distintos se comuniquem entre si, servindo como a ponte técnica que as
    ferramentas ou o MCP utilizam para acessar serviços externos.
  </Accordion>
</AccordionGroup>

### Existem dois tipos de agentes na MomAI

1. **Agentes de delegação**

   ```mermaid
     flowchart TB
         n1("Usuário") --> n3("MomAgent")
         n3 --> n4("SearchAgent") & n5("InterfaceAgent") & n6("SchedulerAgent")
         n4 --> n3
         n5 --> n3
         n6 --> n3
   ```

   Quando o usuário diz algo, MomAgent decide qual ou quais agentes são melhores para determinada tarefa:

   - **InterfaceAgent**: O usuário ao pedir por alguma informação, MomAI se achar necessário usa este agente para criar uma interface mostrando na tela um relatório detalhado.
   - **SchedulerAgent**: O Usuário ao pedir para lembrar ou executar algo em um determinado tempo, data ou hora, MomAI pede para que este agente guarde as informações do agendamento.
   - **SearchAgent**: O usuário ao solicitar uma busca na internet, MomAI passa a solicitação para o agente, ou seja é este que realiza a busca e retorna o resultado para MomAI.

   Extensões, podem adicionar este tipo de agente pensando na especialização em uma determinada tarefa, para expandir as possibilidades de delegação do MomAgent.

2. **Agentes de eventos**

   ```mermaid
   flowchart TB
   n1("Lembrete Acionado") --> n2("ReminderAgent")
   n3("Evento B") --> n4("Agente B")
   ```

   São usados exclusivamente quando um evento é acionado.

   - **ReminderAgent**: Sempre que um evento de lembrete é disparado, este agente envia uma mensagem de voz .

   Ao contrário dos Agentes de delegação que são subordinados e dependem do MomAgent uma vez que o usuário pode pedir qualquer coisa, eventos são previsíveis podendo ser destinados a agentes específicos. Para ficar melhor o entendimento, será que precisa de um gerente cada vez que um cliente entrar na loja? O cliente entrando na loja é um evento previsível e pode ser direcionado diretamente a um atendimento, com os agentes de eventos acontece a mesma coisa, não é obrigatório passar pelo MomAgent pois geralmente eventos possuem uma finalidade específica(Ex: Um evento de agendamento, possui a finalidade de alertar o usuário, logo é o agente de lembretes que vai trabalhar nesse tipo de evento.)

## Um pouco mais sobre eventos

Ao contrário de chatbots onde o usuário envia uma mensagem e espera uma resposta, eventos são acionados quando situações do mundo real acontecem ou seja MomAI pode realizar tarefas de forma autônoma se algum evento for disparado.

### Eventos que já vem por padrão

1. **Scheduler interno** MomAI permite executar uma tarefa em determinado horário, quando chega a hora de realizar a tarefa, é disparado um evento para um agente de eventos (Ex: Todo dia as 07:00 pode falar as principais notícias de tecnologia, e um relatório dos últimos e-mails. Outro exemplo seria lembrar de tomar agua a cada 30 minutos ou a cada 12h verificar o preço de um produto)
2. **Quando o computador inicia**, a assistente inicia junto disparando um evento, que pode ser enviado uma instrução personalizada para qualquer agente

### Alguns eventos que vem ao ativar extensões

1. Quando uma pessoa enviar uma mensagem no WhatsApp, MomAI informa casualmente ao usuário e pergunta se pode responder.

2. Ao ficar muito tempo em redes sociais ou jogando, MomAI alerta os riscos e fala das tarefas que ainda tem que fazer.

   - Avise no grupo dos amigos, que não vou poder ir hoje, é algo relacionado a um projeto importante.
   - Faça uma busca completa sobre Eduardo Mendes e salve em um PDF na minha biblioteca para eu ler depois.
2. Tarefas relacionadas ao computador como:
   - Hibernar, desligar ou reiniciar.
   - Aumentar/ diminuir volume, play/pause.
   - Realizar um comando pelo terminal.
   - Abrir aplicativos e pastas predefinidos pelo usuário em configurações.
   - Criar, editar e deletar arquivos apenas na pasta MomAI workspace
3. Tarefas que exigem memória como:
   - Faça uma lista de vídeos de Python que vi semana passada
   - Não me lembro quais músicas eu vi terça passada, consegue revisar o histórico?
4. Tarefas de agendamentos:
   - Toda terça e quarta fazer um livro caixa em uma planilha do Excel baseado em PDFS da pasta financeiro, fazer um relatório e enviar para um determinado e-mail.

   - Ligue o ar condicionado.
   - Mude a luz da sala para uma cor mais quente

## Python

Então... existe uma alta gama de bibliotecas no NodeJS, no entanto as linguagens JavaScript e TypeScript não são focadas em automatização, tratamento de dados e inteligência artificial, pensando em escalabilidade e simplicidade nas automações, MomAI usa Python no seu núcleo para:

1. Orquestração de agentes usando Langraph/Langchain
2. TTS usando Whisper
3. STT usando Kokoro, OpenAI, GenAI
4. Sistema de extensões
5. Carregamento de modelos locais

## Extensões

O diferencial do projeto é o poder escolher os "poderes" que sua assistente terá acesso, por ser de uso geral algumas funcionalidades poderiam ocupar muito armazenamento e processamento do seu computador, por isso apenas alguns agentes são instalados, enquanto o restante você pode instalar por meio de extensões. Algumas a serem desenvolvidas:

### WhatsApp

Uma integração experimental do MomAI com o Whatsapp. Nota: Devido à complexidade de infraestrutura (Docker/API), esta extensão é recomendada para usuários avançados.

**Requisitos**

- Docker desktop
- Um número cadastrado no Whatsapp

**Assets**

- Adiciona um banco de dados com contatos
- Adiciona uma interface de configuração para:
  - Gerenciar contatos
- Adiciona um arquivo docker compose com evolutionAPi e redis
- Adiciona um arquivo de inicialização do docker.

**Agentes de delegação**

- Agente A: Envia uma mensagem personalizada. Tools:
  - Envia uma mensagem personalizada para um grupo
  - Envia uma mensagem personalizada para um contato

**Eventos**

- Evento A: É emitido quando um contato manda uma mensagem.
- Evento B: Usuário permitiu a resposta

**Agentes de eventos**

- Agente A: Escreve uma frase personalizada sobre a mensagem. Por exemplo "Senhor, você já almoçou? Sua mãe está te perguntando no Whats, posso responder?"
- Agente B: Envia uma mensagem com a resposta.



### Abertura de aplicativos

O usuário pode adicionar uma lista de aplicativos para que MomAI tenha acesso quando solicitado (Ex: Abra a calculadora e o Chrome)

### Navegação

O usuário pode solicitar tarefas em sites específicos. O MomAI usará agentes de navegação (via browser-use) para interagir com a página, clicar em botões e extrair dados ou realizar ações (Ex: Acesse o GitHub e crie um novo repositório).

### Interação com planilhas

Adiciona um agente de delegação com ferramentas para interagir com planilhas presentes no workspace.

### Integração de notas (Anytype, Obsidian, Notion)

Adiciona um agente com acesso a MCP de ferramentas de anotação. Quando o usuário pedir para anotar algo, será feito no aplicativo de anotação padrão.
