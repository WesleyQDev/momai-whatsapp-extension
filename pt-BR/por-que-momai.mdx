## Propósito

inspirado em J.A.R.V.I.s do filme Homem de ferro, MomAI é opensource e multiplataforma, possui a capacidade de: Captar informações e armazenar em um banco de dados local; Executar tarefas quando um evento acontecer; Agendar e criar lembretes; Interagir com o sistema operacional e outros dispositivos; Escutar, falar e até enxergar.
<Warning>
 Os dados são armazenados localmente e processadas pelos provedores de LLM. Cabe ao usuario escolher seus provedores, seja local ou API keys de terceiros.
</Warning>
### Arquitetura

MomAI usa uma arquitetura agentica, MomAgent é o agente principal que verifica a solicitação do usuario, enquanto os demais agentes são especializados em tarefas especificas, como busca na internet, criaçao de agendamentos ou criação de interfaces.

Quando o usuario fala, primeiro é analisado por MomAgent, ele atua como um controlador e supervisor dos demais agentes, você pode pensar como um gerente que delega tarefas aos funcionarios. Lembre-se disso, cada agente possui um ou mias LLM Providers, prompts, ferramentas e MCPs. Com essa arquitetura é possivel desenvolver agentes que possuam ferramentas que se conectam praticamente a qualquer serviço MCP ou API, esses agentes adicionais são disponibilizados por meio de extensões.

### Existem dois tipos de agentes na MomAI

1. **Agentes de delegação**

   ```mermaid
     flowchart TB
         n1("Usuario") --> n3("MomAgent")
         n3 --> n4("SearchAgent") & n5("InterfaceAgent") & n6("SchedulerAgent")
         n4 --> n3
         n5 --> n3
         n6 --> n3
   ```

   Quando o usuário diz algo, MomAgent decide qual ou quais agentes são melhores para determinada tarefa:
   - **InterfaceAgent**: O usuario ao pedir por alguma informação, MomAI se achar necessario usa este agente para criar uma interface mostrando na tela um relatório detalhado.
   - **SchedulerAgent**: O Usuario ao pedir para lembrar ou executar algo em um determinado tempo, data ou hora, MomAI pede para que este agente guarde as informações do agendamento.
   - **SearchAgent**: O usuario ao solicitar uma busca na internet, MomAI passa a solicitação para o agente, ou seja é este que realiza a busca e retorna o resultado para MomAI.

   Extensões, podem adicionar este tipo de agente pensando na especialização em uma determinada tarefa, para expandir as possibilidades de delegação do MomAgent.
2. **Agentes de eventos**

   ```mermaid
   flowchart TB
   n1("Lembrete Acionado") --> n2("ReminderAgent")
   n3("Evento B") --> n4("Agente B")
   ```

   São usados exclusivamente quando um evento é acionado.
   - **ReminderAgent**: Sempre que um evento de lembrete é disparado, este agente envia uma mensagem de voz .

   Ao contrario dos Agentes de delegação que são subordinados e dependem do MomAgent uma vez que o usuario pode pedir qualquer coisa, eventos são previsiveis podendo ser destinados a agentes especificos. Para ficar melhor o entendimento, sera que precisa de um gerente cada vez que um cliente entrar na loja? O cliente entrando na loja é um evento previssivel e pode ser direcionado diretamente a um atendimento, com os agentes de evento acontece a mesma coisa, não é obrigatório passar pelo MomAgent pois geralmente eventos possuem uma finalidade especifica(Ex: Um evento de agendamento, possui a finalidade de alertar o usuario, logo é o agente de lembretes que vai trabalhar nesse tipo de evento.) 


## Um pouco mais sobre eventos

Ao contrário de chatbots onde o usuário envia uma mensagem e espera uma resposta, eventos são acionados quando situações do mundo real acontecem ou seja MomAI pode realizar tarefas de forma autônoma se algum evento for disparado.

### Eventos que já vem por padrão

1. **Scheduler interno** MomAI permite executar uma tarefa em determinado horário, quando chega a hora de realizar a tarefa, é disparado um evento para um agente de eventos (Ex: Todo dia as 07:00 pode falar as principais notícias de tecnologia, e um relatório dos últimos e-mails. Outro exemplo seria lembrar de tomar agua a cada 30 minutos ou a cada 12h verificar o preço de um produto)
2. **Quando o computador inicia**, a assistente incia junto disparando um evento, que pode ser enviado uma instrução personalizada parqa qualquer agente

### Alguns eventos que vem ao ativar extensões

1. Quando uma pessoa enviar uma mensagem no WhatsApp, MomAI informa casualmente ao usuário e pergunta se pode responder.
2. Ao detectar uma pessoa pelo reconhecimento facial e que esteja previamente cadastrada, MomAI fala uma mensagem de boas vindas.
3. Ao ficar muito tempo em redes sociais ou jogando, MomAI alerta os riscos e fala das tarefas que ainda tem que fazer.

## Python

Então... existe uma alta gama de bibliotecas no NodeJS, no entatno as linguagens JavaScript e TypeScript não são focadas em automatização, tratamento de dados e inteligência artificial, pensando em escalabilidade e simplicidade nas automações, MomAI usa Python no seu núcleo para:

1. Orquestração de agentes usando Langraph/Langchain
2. TTS usando Whisper
3. STT usando Kokoro,
4. Sistema de extensões

## Extensões

O diferencial do projeto é o poder escolher os "poderes" que sua assistente tera acesso, por ser de uso geral algumas funcionalidades poderiam ocupar muito armazenamento e processamendo do seu computador, por isso apenas alguns agentes são instalados o restatnte você pode instalar por meio de extensões. Algumas futuras extensões:

1. **WhatsApp**: Adiciona um banco de dados de contatos, um evento é emitido quando um contato manda uma mensagem e um agente fala uma frase personalizada sobre a mensagem. Por exemplo "Senhor, você já almoçou? Sua mãe está te perguntando no Whats, posso responder?"
2. **Reconhecimento facial**: Adiciona o poder de efetuar certas ações caso uma pessoa seja reconhecida, como uma mensagem de boas vindas, um relatório do ocorrido, ou até eventos que passaram enquanto o usuário estava fora do computador.
3. **Abertura de aplicativos**: O usuário pode adicionar uma lista de aplicativos para que MomAI tenha acesso quando solicitado (Ex: Abra a calculadora e o Chrome)
4. **Navegação**: O usuário pode solicitar qualquer coisa que possa ser feito no navegador, MomAI ira tentar finalizar a tarefa, gravara um vídeo passo a passo, e retornara um relatório do ocorrido(Ex: Acesse o GitHub e crie um novo repositório, escolha o nome e a descrição).
5. **Interação com planilhas**: Adiciona um agente de delegação com ferramentas para interagir com planilhas presentes no workspace.

### Extensões da comunidade

Você pode fazer um clone do modelo de exemplo disponibilizado no GitHub para fazer a sua própria extensão. Para publicar na lista da comunidade, basta fazer um fork do arquivo community-plugins.json, adicionar no final da lista as informações do seu plugin e fazer um PR.
