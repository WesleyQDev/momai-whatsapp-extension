## Inspiração

MomAI é inspirado em J.A.R.V.I.s uma IA com acesso a praticamente tudo que é tecnologia criado por Tony Stark no filme homem de ferro.

MomAI consegue lembrar, realizar e agendar tarefas, interagir com dispositivos, falar, escutar e até enxergar, sendo que os dados podem ser processados localmente, MomAI não se mantem conectada a um servidor de forma direta ou seja é de responsabilidade do usuário escolher os provedores que irão tratar os seus dados.

### Arquitetura

MomAI usa uma arquitetura de agentes para quebrar as responsabilidades da assistente, isso também favorece a personalização de prompts, cada agente possuí ferramentas próprias, por padrão existem 5 agentes no entanto as extensões adicionam seus próprios agentes, suas próprias ferramentas e seu próprio prompt.

> Ferramenta nada mas é do que fornecer uma função Python para um LLM e descrever as entradas que essa função precisa para realizar a tarefa.

### Existem dois tipos de agentes

1. **User Agents** - O usuário tem acesso direto as ações do agente. (Ex: Agente de busca)

   Para os User Agents um agente supervisor consegue interagir com esses agentes e quando o usuário pede algo, o supervisor decide qual agente é o melhor para determinada tarefa, o supervisor também pode decidir que a tarefa solicitada seja feita por mais de um agente.
2. **Events Agents** - São usados exclusivamente pelos eventos. (Ex: Agente de execução dos agendamentos)

   Para os Events Agents não há supervisor, extensões que usam IA para o tratamento de dados devem obrigatoriamente especificar um ou mais Events Agents para a execução.

### Orquestração dos agentes

MomAI usa Langraph para a orquestração dos LLMs, em configurações é possível personalizar cada agente disponível e definir um provedor incluindo das extensões. Para a segurança dos seus dados recomendamos usar provedores locais em agentes que executam tarefas críticas ou confidenciais, pois um agente de navegador por exemplo pode ao fazer login deixar exposto senhas ou informações privadas que logo são analisadas pelo seu provedor de LLM, o projeto ainda é experimental e voltada para o uso real, sugestões para o aprimoramento da segurança são sempre bem vindas!

## Eventos

Ao contrário de chatbots onde o usuário obrigatoriamente tem que falar um comando para que um LLm responda, MomAI é amplamente movido a eventos. Sendo que pode realizar tarefas de forma autônoma desde que algum evento previamente autorizado seja acionado.

### Eventos que já vem por padrão

1. **Scheduler interno** que permite a MomAI executar uma tarefa em determinado horário (Ex: Todo dia as 07:00 pode falar as principais notícias de tecnologia, e um relatório dos últimos e-mails. Outro exemplo seria lembrar de tomar agua a cada 30 minutos ou a cada 12h verificar o preço de um produto)
2. **Quando o computador inicia**, MomAI pode iniciar junto e falar uma mensagem de inicialização ao usuário.

### Alguns eventos que vem ao ativar extensões

1. Quando uma pessoa enviar uma mensagem no WhatsApp, MomAI informa casualmente ao usuário e pergunta se pode responder.
2. Ao detectar uma pessoa pelo reconhecimento facial e que esteja previamente cadastrada, MomAI fala uma mensagem de boas vindas.
3. Ao ficar muito tempo em redes sociais ou jogando, MomAI alerta os riscos e fala das tarefas que ainda tem que fazer.

## Possibilidades ao usar MomAI + Extensões

1. Pedir para realizar tarefas no geral como:
   - Faça um relatório baseado em informações da internet, Youtube, Gmail, Google agendas, Notion, Obsidian.
   - Qual a previsão do tempo para hoje?
   - Estou entediado consegue me fornecer uma lista com principais canais do Youtube que citaram Python em seus últimos vídeos?
   - Abra o sistema da faculdade e me diga as atividades que ainda tenho que fazer.
   - Consegue avisar no grupo que não vou poder ir hoje?
   - Quero que me diga tudo o que você sabe sobre Eduardo Mendes e salve em um PDF para eu ver depois.
2. Tarefas relacionadas ao computador como:
   - Hibernar, desligar ou reiniciar.
   - Aumentar/ diminuir volume, play/pause.
   - Realizar um comando pelo terminal.
   - Abrir aplicativos e pastas predefinidos pelo usuário em configurações.
   - Criar, editar e deletar arquivos apenas na pasta MomAI workspace
3. Tarefas que exigem memória como:
   - Você consegue fazer uma lista de vídeos de Python que vi semana passada?
   - Você se lembra o que eu comi terça passada? Não me lembro se falei para você
4. Tarefas de agendamentos:
   - Toda terça e quarta fazer um livro caixa em uma planilha do Excel baseado em PDFS da pasta financeiro, fazer um relatório e enviar para um determinado e-mail.
   - Me lembre daqui 3 minutos de jogar o lixo fora.
   - Todo 1º de abril, quando detectar alguém faça uma piada de pane no sistema.
5. Tarefas relacionados a automatização residencial como:
   - Ligue o ar condicionado.
   - Mude a luz da sala para uma cor mais quente
   - Sempre que você me detectar no reconhecimento facial pisque as luzes.

> Futuramente MomAI recebera APP mobile e tera acesso a ferramentas para controlar o celular pelo pc e controlar o pc pelo celular, inclúsivel é possivel na maioria dos computadores ligar pelo própio celular usando wake on lan, para maior segurança o APP se conectara apenas a um wbsoket fornecido pelo FastAPI local. Novamente resaltando, este é um projeto experimental, priorizamos a segurança no entanto features serão lançadas constantemente e ainda não há uma previsibilidade para lançar a primeira versão estavel.

## Python

Por mas que exista uma alta gama de bibliotecas no NodeJS, as linguagens JavaScript e TypeScript não são focadas em automatização, tratamento de dados e inteligência artificial, pensando em escalabilidade e simplicidade nas automações, MomAI usa Python no seu núcleo para:

1. Orquestração de agentes usando Langraph/Langchain
2. TTS usando Whisper
3. STT usando Kokoro,
4. Ferramentas por defeito
5. Extensões

## Extensões

O diferencial da MomAI é o poder escolher os poderes que você quer que sua Assistente possua, por padrão MomAI já fornece muitas funcionalidades no entanto você pode optar por adicionar extensões, aqui estão alguns exemplos de extensões:

1. WhatsApp: Uma poderosa extensão ao qual quando um contato manda uma mensagem o assistente alerta o usuário com uma mensagem personalizada como por exemplo "Wesley, você já almoçou? Sua mãe está te perguntando no Whats, posso responder?"
2. Reconhecimento facial: Adiciona o poder de efetuar certas ações caso uma pessoa seja reconhecida, como uma mensagem de boas vindas, um relatório do ocorrido, ou até eventos que passaram enquanto o usuário estava fora do computador.
3. Abrir aplicativos: O usuário pode adicionar uma lista de aplicativos para que MomAI tenha acesso quando solicitado (Ex: Abra a calculadora e o Chrome)
4. Navegador: O usuário pode solicitar qualquer coisa que possa ser feito no navegador, MomAI ira tentar finalizar a tarefa, gravara um vídeo passo a passo, e retornara um relatório do ocorrido(Ex: Acesse o GitHub e crie um novo repositório, escolha o nome e a descrição).
5. Excel

### Extensões da comunidade

Você pode fazer um clone do modelo de exemplo disponibilizado no GitHub para fazer a sua própria extensão. Para publicar na lista da comunidade, basta fazer um fork do arquivo community-plugins.json, adicionar no final da lista as informações do seu plugin e fazer um PR.

## Referências e inspirações de outros projetos

1. [vocalis](https://github.com/shaakz/vocalis) É um assistente com capacidades de speak to speak, usa Reactjs com backend FastAPI. Parte do código da escuta e da fala foram inspirados nesse projeto.
2. [Obsidian](https://obsidian.md/) Seus plugins são de software aberto e desenvolvidos pela comunidade, o aplicativo em si é free use e auto hospedado, MomAI foi construída se inspirando um pouco no modelo de plugins do Obsidian.
