O sistema de extensões da MomAI segue a filosofia do Obsidian: pastas autocontidas com um manifesto e um ponto de entrada Python.

## Estrutura de uma Extensão
Toda extensão deve residir em uma subpasta dentro de `apps/core/extensions/` (para oficiais) ou na pasta de dados do usuário (`%APPDATA%/MomAI/extensions/`).

```text
my-extension/
├── manifest.json      # Metadados e permissões
└── plugin.py          # Lógica Python e Hooks
```

## 1. O Manifesto (`manifest.json`)
Define quem você é, o que sua extensão faz e quais permissões ela exige.

```json
{
  "id": "my-extension-id",
  "name": "Meu Plugin",
  "author": "WesleyQDev",
  "version": "1.0.0",
  "description": "Faz algo incrível.",
  "icon": "Cpu", // Nome do ícone no Heroicons
  "intents": [
    "Minha intenção secreta",
    "Acione meu comando"
  ],
  "features": {
    "sidebar": true,
    "agent": "SystemAgent" // Para qual agente as intenções devem ir
  },
  "permissions": {
    "network": ["api.meusite.com"],
    "system": ["hardware_read"]
  }
}
```

## 2. Implementação com Hooks (`plugin.py`)
Utilizamos o `pluggy` para que a extensão se conecte ao núcleo.

```python
from langchain_core.tools import tool
from services.extensions.hooks import hookimpl

@tool
def minha_ferramenta_nova(query: str):
    """Descrição clara para o Tool RAG encontrar esta função."""
    return f"Resultado para {query}"

@hookimpl
def register_tools():
    """Registra ferramentas no Tool RAG global."""
    return [minha_ferramenta_nova]

@hookimpl
def register_sidebar_items():
    """Adiciona um ícone na barra lateral do React."""
    return [{
        "id": "minha-view",
        "icon": "Cpu",
        "label": "Minha Extensão",
        "view": "MinhaViewCustomizada"
    }]
```

## Ciclo de Inteligência
Ao criar uma extensão, você não precisa configurar o modelo de linguagem manualmente:
1. O **Semantic Router** lê suas `intents` do manifesto e as indexa no LanceDB.
2. O **Tool RAG** lê as descrições das suas ferramentas decoradas com `@tool`.
3. Quando o usuário pede algo relacionado, a MomAI automaticamente direciona o fluxo para sua extensão e vincula suas ferramentas ao LLM.

## Segurança e Transparência
As extensões não rodam em sandbox, mas o `ExtensionManager` realiza uma auditoria estática:
- Se seu código importar `requests` ou `socket`, mas você não declarou a permissão `network` no manifesto, o MomAI emitirá um alerta de segurança e poderá bloquear o carregamento.
- Plugins oficiais (`author: "WesleyQDev"`) são carregados automaticamente. Plugins de terceiros exigem que o usuário desative o "Safe Mode".
