O backend da MomAI é uma aplicação **FastAPI** robusta, projetada para ser o sistema operacional da sua inteligência artificial pessoal.

## Pilares Técnicos

### 1. Orquestração com LangGraph
Diferente de fluxos lineares, usamos o **LangGraph** para criar um Grafo de Estados. Isso permite que a assistente "pense" em ciclos: ela pode usar uma ferramenta, observar o resultado e decidir se precisa de outra ferramenta antes de responder.

- **Checkpoints**: Utilizamos SQLite (`checkpoints.db`) com o motor `AsyncSqliteSaver` para persistir o estado do grafo. Isso garante que a assistente mantenha o contexto mesmo após reinicializações.
- **Nós de Decisão**: O fluxo é decidido dinamicamente. Se um comando exige acesso a arquivos, o orquestrador seleciona as ferramentas e o agente específico sem intervenção manual.

### 2. Memória Vetorial e Roteamento (LanceDB)
O **LanceDB** atua como nossa infraestrutura de RAG (Retrieval-Augmented Generation) e roteamento:
- **Semantic Router**: Com base em embeddings locais, a MomAI mapeia o que você disse para intenções pré-indexadas.
- **Tool RAG**: Em vez de passar todas as ferramentas para o LLM (o que gastaria muitos tokens), consultamos o LanceDB para extrair apenas as ferramentas que fazem sentido para o contexto atual.

### 3. Sistema de Extensões Dinâmico
Utilizamos o `pluggy` para garantir que o núcleo permaneça "limpo" enquanto as funcionalidades são injetadas via extensões. O `ExtensionManager` é responsável por:
- Carregar os manifestos e registrar as ferramentas no banco vetorial.
- Injetar itens de UI na barra lateral do frontend.
- Gerenciar os ambientes virtuais (venvs) específicos de cada plugin.

## Estrutura de Dados (SQLite)

O sistema utiliza o SQLite para armazenamento relacional e persistente em `momai.db`:
- **Settings**: Todas as preferências de voz, temas e chaves de API.
- **Messages**: Histórico de chat para renderização visual rápida.
- **GamingApps**: Lista de gatilhos para o motor FortScript.
- **Reminders**: Agendamentos e notificações do sistema.

---

## Fluxo de Inicialização (Lifespan)

Ao iniciar o backend, os seguintes passos são executados na ordem:
1. **Migrations**: O esquema do banco SQLite é validado e atualizado.
2. **Extensions Registry**: O sistema varre as extensões locais e remotas.
3. **Internal Indexing**: Ferramentas e agentes são indexados no LanceDB baseando-se nos manifestos.
4. **Resorce Broadcaster**: Inicia a thread de monitoramento de hardware (CPU/RAM/VRAM).
5. **Wake Word Engine**: O motor **Vosk** é carregado e começa a ouvir em background.
6. **FortScript Manager**: O monitor de processos pesados entra em ação.

## Endpoints de Integração

- **`POST /chat/stream`**: Endpoint principal de interação que retorna um `StreamingResponse` com tokens e eventos do grafo.
- **`GET /system/gaming-apps`**: Gerenciamento dos apps monitorados pelo FortScript.
- **`WS /ws`**: Socket bidirecional para eventos de sistema, notificações em tempo real e telemetria de hardware.
- **`GET /setup/status`**: Verifica o estado dos motores locais (Llama.cpp, TTS) e sugere instalações.
