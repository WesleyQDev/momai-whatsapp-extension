O backend da MomAI é uma aplicação **FastAPI** rodando em **Python 3.12**, que atua como o orquestrador entre o hardware local, provedores de nuvem e a interface do usuário.

## Pilares Técnicos

### 1. Orquestração com LangGraph
Diferente de fluxos lineares, usamos o **LangGraph** para criar um Grafo de Estados. Isso permite que a assistente "pense" em ciclos: ela pode usar uma ferramenta, observar o resultado e decidir se precisa de outra ferramenta antes de responder.

- **Checkpoints**: Utilizamos SQLite (`checkpoints.db`) para persistir o estado do grafo. Isso permite que a assistente lembre do que estava fazendo mesmo após reiniciar.
- **Nós Dinâmicos**: O nó `agent_node` é genérico e injeta ferramentas em tempo real via Tool RAG.

### 2. Memória Vetorial (LanceDB)
O **LanceDB** é o nosso banco de dados vetorial local e serverless. Ele é usado para:
- **Semantic Router**: Mapear a frase do usuário para o agente correto (confiança baseada em distância L2).
- **Tool RAG**: Filtrar as ferramentas mais relevantes entre centenas de opções disponíveis em extensões.

### 3. Sistema de Extensões (`pluggy`)
Utilizamos o `pluggy` (o mesmo motor do `pytest`) para gerenciar plugins. O `ExtensionManager` carrega módulos dinamicamente e consolida ferramentas e itens de UI.

## Fluxo de Inicialização (Lifespan)
1. **Migrations**: Verifica e atualiza o esquema do banco de dados SQLite.
2. **Extensions**: O `ExtensionManager` varre as pastas e registra os hooks.
3. **Indexing**: O `indexer.py` atualiza os vetores de ferramentas e intenções no LanceDB.
4. **Wake Word**: Inicia o detector **Vosk** em uma thread separada.
5. **LLM**: Inicializa o provedor padrão (Local ou Cloud).

## Endpoints Chave
- `POST /chat/stream`: Stream de eventos (tokens, status, abertura de grafos).
- `GET /extensions`: Lista manifestos das extensões ativas.
- `WS /ws`: Canal bidirecional para telemetria de hardware e eventos em tempo real.
