A MomAI não é um chatbot simples; ela é um ecossistema de agentes coordenados. Utilizamos o **LangGraph** para criar fluxos de trabalho complexos, cíclicos e resilientes.

## O Fluxo de Decisão (O Ciclo de Inteligência)

Quando você envia uma mensagem, ela passa por várias etapas de filtragem e decisão:

### 1. Roteador Semântico (LanceDB)
O primeiro passo é totalmente local. Usamos o **LanceDB** para comparar sua mensagem com uma base de dados de intenções pré-indexadas via embeddings reais.
- **Vantagem:** Se você diz "Que horas são?", o sistema identifica instantaneamente que deve chamar o `SystemAgent` sem precisar gastar tokens em um LLM.
- **Confiança:** Se a similaridade for baixa ou o pedido for ambíguo, ele é enviado ao Orquestrador Estratégico.

### 2. O Loop de Execução do Grafo
Uma vez que o contexto é direcionado para um agente, ele entra em um loop `Action -> Observe -> Decide`:

1.  **Nó Orquestrador**: Recebe o histórico de mensagens e as ferramentas injetadas pelo Tool RAG. Decide se precisa acionar uma ferramenta ou se pode responder diretamente.
2.  **Nó de Ferramentas (Tools)**: Se o orquestrador decidir por uma ação (ex: `read_file`), este nó executa o código Python real e devolve o resultado para o grafo.
3.  **Nó Responder**: É o estágio final. Ele pega todas as observações das ferramentas e redige a resposta final para você, garantindo que o tom de voz "Senhor" e a personalidade sejam mantidos.

### 3. Tool RAG (Retrieval Augmented Tools)
Diferente de outros assistentes que tentam enviar todas as funções disponíveis para o LLM, a MomAI usa **Tool RAG**. 
- O sistema busca no banco vetorial as ferramentas mais relevantes para o contexto atual entre todas as extensões instaladas.
- Isso permite que a MomAI tenha milhares de ferramentas disponíveis através de plugins de terceiros sem comprometer a janela de contexto do LLM.

## Memória e Estado Persistente

Utilizamos o `AsyncSqliteSaver` do LangGraph para manter o estado da jornada do usuário. 
- **Thread Management**: Cada conversa tem um `thread_id`. O orquestrador recupera o estado exato da conversa anterior do `checkpoints.db`.
- **Continuidade de Tarefa**: Se você pedir para a IA baixar um arquivo grande e a internet cair, ela lembrará em que passo da tarefa estava na próxima interação.
