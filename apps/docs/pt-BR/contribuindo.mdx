## Configurando o Ambiente

<Steps>
  <Step title="Fork e Clone">
    ```bash
    # Fork no GitHub, depois:
    git clone https://github.com/SEU-USUARIO/MomAI.git
    cd MomAI
    ```
  </Step>

<Step title="Instalar Dependências Frontend">
  ```bash # Requer Node.js 20+ e pnpm pnpm install ```
</Step>

  <Step title="Configurar Backend Python">
    ```bash
    cd apps/core
    python -m venv .venv

    # Windows
    .venv\Scripts\activate
    # Linux/macOS
    source .venv/bin/activate

    pip install -e ".[dev]"
    ```

  </Step>

  <Step title="Executar Testes">
    ```bash
    # Backend
    cd apps/core
    pytest

    # Frontend
    cd apps/desktop
    pnpm test
    ```

  </Step>
</Steps>

## Estrutura do Projeto

```
MomAI/
├── apps/
│   ├── core/           # Backend Python
│   │   ├── ai/         # Orquestração, LLM, embeddings
│   │   ├── database/   # SQLite, LanceDB
│   │   ├── services/   # Extensões, voz, sistema
│   │   ├── tools/      # Ferramentas do núcleo
│   │   └── agents/     # Agentes nativos e extensões
│   ├── desktop/        # Frontend Electron + React
│   │   └── src/
│   │       ├── main/       # Processo principal Electron
│   │       ├── preload/    # Bridge de APIs
│   │       └── renderer/   # Aplicação React
│   ├── docs/           # Esta documentação (Mintlify)
│   └── fortscript/     # Biblioteca de gerenciamento de recursos
└── packages/           # Pacotes compartilhados
```

## Guia de Estilo

### Python (Backend)

- **Formatter:** Ruff
- **Linter:** Ruff
- **Type Hints:** Obrigatório em funções públicas
- **Docstrings:** Google style

```python
def processar_mensagem(texto: str, contexto: dict | None = None) -> str:
    """Processa uma mensagem do usuário.

    Args:
        texto: Mensagem do usuário.
        contexto: Contexto adicional da conversa.

    Returns:
        Resposta processada.

    Raises:
        ValueError: Se o texto estiver vazio.
    """
    if not texto:
        raise ValueError("Texto não pode estar vazio")
    ...
```

### TypeScript (Frontend)

- **Formatter:** Prettier
- **Linter:** ESLint
- **Framework:** React 19 com hooks

```tsx
interface ChatMessageProps {
  content: string;
  role: "user" | "assistant";
  timestamp: Date;
}

export function ChatMessage({ content, role, timestamp }: ChatMessageProps) {
  return (
    <div className={`message ${role}`}>
      <p>{content}</p>
      <time>{timestamp.toLocaleTimeString()}</time>
    </div>
  );
}
```

## Fluxo de Contribuição

<Steps>
  <Step title="Criar Branch">
    ```bash
    git checkout -b feature/minha-feature
    # ou
    git checkout -b fix/meu-bugfix
    ```
  </Step>

<Step title="Desenvolver">
  Implemente sua mudança seguindo os guias de estilo. Adicione testes quando
  aplicável.
</Step>

  <Step title="Testar">
    ```bash
    # Backend
    cd apps/core
    ruff check .
    pytest

    # Frontend
    cd apps/desktop
    pnpm lint
    pnpm test
    ```

  </Step>

<Step title="Commit">
  Use commits semânticos: ```bash git commit -m "feat: adicionar suporte a
  lembretes recorrentes" git commit -m "fix: corrigir crash ao carregar
  extensões" git commit -m "docs: atualizar guia de instalação" ```
</Step>

  <Step title="Pull Request">
    1. Push para seu fork
    2. Abra PR para `main`
    3. Descreva as mudanças
    4. Aguarde review
  </Step>
</Steps>

## Convenções de Commit

| Prefixo     | Uso                                |
| ----------- | ---------------------------------- |
| `feat:`     | Nova funcionalidade                |
| `fix:`      | Correção de bug                    |
| `docs:`     | Documentação                       |
| `style:`    | Formatação (sem mudança de código) |
| `refactor:` | Refatoração                        |
| `test:`     | Testes                             |
| `chore:`    | Manutenção geral                   |
